Here is the next big feature that I want to add: Exporting to PDF

Multiple Elements can be exported to pdf:
1. Sections of the reading view
2. Notes Pages
3. Outlines

Reguardless of how the user gets there or what they are exporting, the popup for exporting looks the same.
The popup will have a preview of what the exported PDF will look like on the left, and on the right of that the PDF options similar to how a print or regular export to PDF screen would look like appear. At the bottom of the export options side of the page is a button for setting the download path and a button for exporting the PDF. In review:
- Left side: Preview of the PDF
- Right side: PDF options (depends on the type of file being exported)
- Bottom of right side: Download path and export button

The user can get to the export menu by either going to a button under file > export > (sub options) or hitting "ctrl+p"
If the user navigates to the submenu, the menu will have options to export for each of the types of documents open in the central view. So if the user is in a split screen view with a Notes page and a Reading view, they should have an option to either Export Note to PDF (Or Export Notes to .docx but more about that later) or Export Passage to PDF.

hitting ctrl+p will open a popup menu with the same options as the submenu.

## Sections of the Reading View
- verse range of the passage to export
- Toggle if Book heading should be included (default to yes)
- Toggle if chapter headings should be included (default to yes)
- Toggle if verse numbers should be included (default to yes)
- Button to open a menu to select what types of marks should be included (default to all)
- Button to open a menu to select what types of symbols should be included AS WELL AS, what color to make them. (default to all with a neutral grey that could be seen on black or white paper)
- Line Spacing
- Font Size
- Page Orientation
- Margins

All of these options are used to make the PDF that is generated. The reading scene is usually a dark background with white text but that will change on export to PDF which will need to be white background on black text

## Notes Pages
Notes pages can be exported almost as is, except for changing the background from dark to white and the text from white to black (if the user has defined their own color options for text dont color swap that). There should be a three dot icon at the top right of a notes.

Notes can also be exported to .docx as well as pdf. This should be an option in the submenus (file > export > (sub options) or ctrl+p).


## Outlines
Outlines are very similar to notes pages in that they can be exported to .docx as well as pdf and should default to a white background and black text reguardless of the users color options for the reading view.
